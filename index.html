§0 What is Statistics?

Statistics is the science of collecting, organizing, summarizing, and analyzing information to draw conclusions or answer questions. It is not an exact science — it is a discipline grounded in inference and uncertainty.

In computer science specifically, statistics is the discipline of structuring datasets and applying quantitative methods to build models that extract patterns, support prediction, and enable data-driven decision-making.

Two Main Branches

Branch 1
Descriptive Statistics
Focuses on presenting data using numerical summaries and visual tools. Its purpose is to highlight patterns and provide clear insights into the dataset. It describes what the data is — it does not generalize beyond the data you have.
Branch 2
Inferential Statistics
Uses sample data to draw conclusions, make predictions, or generalize findings about a larger population. It goes beyond what you can directly see — it estimates what is likely true for the whole group.

Population vs. Sample

One of the most important distinctions in statistics is between the full group you care about and the subset you actually study.

Key Concept
Population (N)
The entire set of individuals, items, or data points of interest in a study. Studying the full population is often impractical — too large, too expensive, or impossible.

Example: All PUP students enrolled this semester.
Key Concept
Sample (n)
A subset of the population selected for analysis, used to make inferences about the whole. A well-chosen sample can represent the population accurately.

Example: 200 randomly selected PUP students from that population.
📌 The Inference CycleWe use probability to draw a representative sample from the population → we analyze the sample with descriptive statistics → we use statistical inference to make conclusions about the population. The direction flows both ways: probability governs sampling; inference interprets the results.

§1 Datasets & Variable Classification

Before any analysis can begin, you need to understand the structure of your dataset and the nature of each variable it contains. This step is the foundation of Exploratory Data Analysis (EDA).

Dataset Anatomy

A dataset is a systematically organized group of related data points, arranged in a way that makes them easy to access, analyze, and process. Every dataset has four core components:

Component 1
Variables
The features or attributes being measured — the columns of your table. Each variable captures one aspect of each observation (e.g., Age, Sex, Credit Amount).
Component 2
Observations
The individual records or entries — the rows of your table. Each observation is one instance of the thing being studied (e.g., one loan applicant, one transaction).
Component 3
Metadata
Information about the data — not the data itself. Includes the source, collection date, number of rows/columns, units of measurement, and usage notes. Provides the context needed to interpret the dataset correctly.
Component 4
Schema
The formal structural blueprint: column names, data types, and relational constraints. Ensures the dataset is stored consistently and can be validated — e.g., "Age: numeric, Sex: text (male/female), Credit amount: numeric in DM."

Exploratory Data Analysis (EDA)

EDA is the systematic approach of examining datasets using descriptive statistics and visualization techniques to understand their structure, relationships, and key characteristics before applying formal modeling or hypothesis testing.

💡 The 4 Steps of EDAAll of Part 1 follows this workflow. Keep it in mind as a roadmap.
  1. Understand the Dataset — Review metadata and schema; identify the dataset's size; classify each variable by type and level of measurement.
  2. Data Cleaning — Handle missing values (drop, impute, or flag); detect and treat outliers; standardize formats (dates, categories, units).
  3. Descriptive Statistics — Univariate analysis (one variable at a time: central tendency, dispersion, shape, visual summaries) and bivariate/multivariate analysis (relationships between two or more variables).
  4. Initial Insights — Summarize key findings; identify potential hypotheses or directions for deeper analysis.

Variable Classification — Qualitative vs. Quantitative

All variables fall into one of two broad categories before we assign a level of measurement:

Category 1
Qualitative (Categorical)
Variables that yield categorical responses — words or codes that represent a class or category. You cannot do arithmetic on them in a meaningful way.

Examples: Sex, Housing type, Purpose, Risk (Good/Bad)
Category 2
Quantitative (Numerical)
Variables that take on numerical values representing an amount or quantity. Arithmetic is valid and meaningful.

Examples: Age, Credit Amount, Duration (months)

Levels of Measurement — The NOIR Scale

Within each category, the level of measurement describes the relationship among the values assigned to a variable's attributes. Each higher level adds one more mathematical property. This is critical because the level determines which statistics and charts are valid.

💡 Memory TipThink of NOIR: Nominal → Ordinal → Interval → Ratio. Each level in the scale adds one more mathematical property to the previous.
Qualitative · NOIR Level 1
Nominal
Categories with no natural order. You cannot say one label is "greater" than another. Arithmetic is meaningless.

Dataset examples: Sex (Male/Female), Risk (Good/Bad), Housing type
CS examples: Operating System Type (Windows, macOS, Linux), Programming Language (Python, Java, C++)
Qualitative · NOIR Level 2
Ordinal
Categories with a meaningful rank. The order is clear, but the gaps between ranks are not necessarily equal — you can't say "Admin is exactly twice as much as User."

Dataset examples: Job classification (0→3), Savings level (none→rich)
CS examples: User Access Level (Guest → User → Admin → Super Admin), Support Ticket Priority (Low → Medium → High → Critical)
Quantitative · NOIR Level 3
Interval
Numbers where zero does not mean "none" — zero is just a point on the scale. Differences are meaningful; ratios are not (0°C does not mean "no temperature").

CS examples: CPU Temperature in °C/°F (40°C, 70°C), Time of day in 24-hour format (13:00, 18:45)
Quantitative · NOIR Level 4
Ratio
Numbers with a true absolute zero. Zero means the complete absence of the quantity. All arithmetic — including ratios — is valid (twice the file size really is twice as large).

Dataset examples: Age, Credit Amount, Duration
CS examples: File size in MB/GB (0 MB = no file), Number of lines of code (0 lines = no code)

§3 Measures of Central Tendency

Central tendency measures describe where the "center" of a dataset lies. There are three: the mean, median, and mode.

Standard Notation (used throughout this section) x₁, x₂, …, xₙ — Individual observations in a dataset
N, n — Number of observations in the population and sample, respectively
μ (mu) — Population mean
x̄ (x-bar) — Sample mean
x̃ (x-tilde) — Sample median (middle value of ordered observations)
x* — Mode (the observation with the highest frequency)

Mean (Arithmetic Average)

Add all values, divide by how many there are. The most common "average" in everyday life.

Mean Formula x̄ = (x₁ + x₂ + ··· + xₙ) / n = Σxᵢ / n
Worked Example
Ages of 5 loan applicants: 22, 25, 30, 35, 38. Find the mean.
Sum = 22 + 25 + 30 + 35 + 38 = 150
n = 5
x̄ = 150 / 5 = 30 years
Lecture Dataset Example
Dataset: 0.30, 0.35, 0.42, 0.45, 0.47, 0.50, 0.53, 0.60, 0.72, 0.85 (n = 10)
Sum = 0.30 + 0.35 + 0.42 + 0.45 + 0.47 + 0.50 + 0.53 + 0.60 + 0.72 + 0.85 = 5.19
x̄ = 5.19 / 10
x̄ = 0.519

Key Properties of the Mean

PropertyWhat happensExample (using x̄ = 0.519)
Multiply all values by k (scaling)Mean multiplies by k: mean({k·xᵢ}) = k·x̄Multiply all by 0.75 → new x̄ = 0.75 × 0.519 = 0.389
Add constant c to all values (translation)Mean increases by c: mean({xᵢ + c}) = x̄ + cAdd 0.05 to all → new x̄ = 0.519 + 0.05 = 0.565
Sum of signed deviations from meanAlways equals 0: Σ(xᵢ − x̄) = 0The mean is the "balance point" of the data
Outlier sensitivityOne extreme value can pull mean far from centerReplace 0.85 with 5.00 → mean shifts dramatically

Median (Middle Value)

The median is the value that splits a sorted dataset exactly in half. It is robust to outliers — extreme values don't move it much.

Finding the Median (x̃) Step 1: Sort data from smallest to largest
Step 2 (odd n): x̃ = value at position (n + 1) / 2
Step 2 (even n): x̃ = average of values at positions n/2 and (n/2 + 1)
Worked Example — Odd n
Data: 12, 15, 18, 22, 30 (n = 5)
Position = (5+1)/2 = 3rd value
Median = 18
Worked Example — Even n (Lecture Dataset)
Dataset 1: 0.30, 0.35, 0.42, 0.45, 0.47 | 0.50, 0.53, 0.60, 0.72, 0.85 (n=10)
5th value = 0.47, 6th value = 0.50
x̃ = (0.47 + 0.50) / 2
Median = 0.485  |  Mean = 0.519  →  Mean > Median (slight right skew from 0.85)
📌 Mean vs. Median — When Does It Matter?Compare these two datasets from the lecture:
Dataset 1: 0.30, 0.35, 0.42, 0.45, 0.47, 0.50, 0.53, 0.60, 0.72, 0.85 → Mean = 0.519, Median = 0.485
Dataset 2: 0.30, 0.35, 0.40, 0.42, 0.45, 0.47, 0.50, 0.53, 0.60, 0.72 → Mean = 0.474, Median = 0.460
Dataset 1 has a high outlier (0.85) that pulls its mean up relative to the median. When data is skewed, the median is a better representative of center.

Mode (Most Frequent Value)

The only measure of central tendency that works for nominal (categorical) data. The mode is the value x* whose frequency is greater than or equal to the frequency of any other value in the dataset.

Modality
Unimodal
One value has the highest frequency. The dataset has a single peak.
Example: {2, 3, 3, 4, 5} → Mode = 3
Modality
Bimodal / Multimodal
Two or more values tie for the highest frequency. The dataset has multiple peaks.
Example: {2, 3, 3, 4, 4, 5} → Modes = 3 and 4
Modality
No Mode
All values occur with equal frequency. The mode is not uniquely defined.
Example: {1, 2, 3, 4, 5} → no mode
Example — Categorical Mode
Risk ratings: Good, Bad, Good, Good, Bad
Mode = Good (appears 3 times)

§4 Dispersion, Shape & Outlier Detection

Measures of Dispersion (How Spread Out Is the Data?)

Range

Range = Maximum − Minimum

Limitation: Uses only two values, so one outlier can completely change it.

Variance and Standard Deviation

Average of the squared distances from the mean. We square so that negative and positive gaps don't cancel out. There are two versions — one for a sample and one for a full population.

Sample Variance & Standard Deviation s² = Σ(xᵢ − x̄)² / (n − 1)
s = √s²
Population Variance & Standard Deviation σ² = Σ(xᵢ − μ)² / N
σ = √σ²
📌 Why (n − 1) for samples?For sample data, dividing by n−1 (instead of n) corrects for bias and gives a better estimate of the true population variance. This is called Bessel's correction. For the full population, you use N because there's no estimation needed.
Worked Example
Data: 2, 4, 4, 4, 5, 5, 7, 9 (n=8). Find the variance and standard deviation. (Mean = 5)
Deviations (xᵢ − 5): −3, −1, −1, −1, 0, 0, 2, 4
Squared: 9, 1, 1, 1, 0, 0, 4, 16 → Sum = 32
Variance: 32 / (8−1) = 32/7 ≈ 4.57
s² ≈ 4.57   |   s = √4.57 ≈ 2.14

Outlier Detection — The IQR Method

Box Plot Boundaries IQR = Q3 − Q1
Lower Fence = Q1 − 1.5 × IQR
Upper Fence = Q3 + 1.5 × IQR

Any value outside [Lower Fence, Upper Fence] is an OUTLIER ●
Worked Example
Data: 100, 200, 300, 400, 500, 600, 5000. Q1=200, Q3=500. Is 5000 an outlier?
IQR = 500 − 200 = 300
Upper Fence = 500 + 1.5(300) = 500 + 450 = 950
5000 > 950 → 5000 IS an outlier

Skewness > 0

Tail extends to the right. A few very high values pull the mean up.

Example: Income data
Skewness < 0

Tail extends to the left. A few very low values pull the mean down.
Mean < Median.

§8 What is Probability?

Probability is the mathematical language for quantifying uncertainty. It tells you how likely an event is to occur, expressed as a number between 0 and 1.

Scale
P = 0
The event is impossible. It will never occur.
Example: Rolling a 7 on a standard 6-sided die.
Scale
P = 0.5
The event is equally likely to happen or not happen.
Example: Getting heads on a fair coin flip.
Scale
P = 1
The event is certain. It will always occur.
Example: Rolling a number ≤ 6 on a standard die.

Why Probability Matters in Statistics

The cycle of statistical inference works like this: we use probability to draw a sample from a population, analyze the sample, then use statistical inference to make conclusions about the population. Probability connects the two — it tells us how reliable our conclusions are.

📌 Key Questions Probability Answers"How reasonable is this outcome?" and "What's the chance our conclusion is wrong?" — these are the foundations of hypothesis testing and confidence intervals you'll encounter later in the course.
§9 Sample Spaces & Events

Sample Space (S)

The sample space is the set of ALL possible outcomes of a random experiment. Every single outcome that could happen must be listed (or described) in S.

Notation S = {ω | ω is all possible outcomes}

Three Ways to Describe a Set

MethodDescriptionExample
DescriptiveShort verbal statement"All outcomes when flipping a coin twice"
Roster / ListingList all elements inside { }S = {HH, HT, TH, TT}
Set-Builder NotationVariable + condition after a vertical bar |S = {x | x is an integer, 1 < x < 10}

Types of Sample Spaces

Type 1
Finite
A limited number of distinct outcomes.

Example: Flipping a coin twice → S = {HH, HT, TH, TT} — only 4 outcomes.
Type 2
Countably Infinite
Outcomes can be listed but never end.

Example: Number of coin flips until first head → S = {1, 2, 3, 4, ...} — could go on forever.
Type 3
Uncountably Infinite
Outcomes form a continuous range — cannot be listed.

Example: Lifespan of an electronic component → S = {t | t ∈ ℝ⁺}

Events

An event is any subset of the sample space. An event A is said to have occurred if the actual outcome of the experiment is an element of A.

Examples
S = {HH, HT, TH, TT} — flipping a coin twice.
Event A (exactly one head): A = {HT, TH}
Event B (no heads): B = {TT}
The sample space S itself is also an event (the "certain" event).
The empty set ∅ is also an event (the "impossible" event).
§10 Operations on Events

Events can be combined or modified using set operations, just like sets in mathematics.

OperationSymbolDefinitionPlain English
ComplementA′All elements of S NOT in A"Everything except A"
IntersectionA ∩ BAll elements in BOTH A and B"A AND B both happen"
UnionA ∪ BAll elements in A OR B or both"A OR B (or both) happens"
Mutually ExclusiveA ∩ B = ∅A and B share NO elements"A and B cannot both happen at the same time"
Full Example — Network Protocol
Sending two packets. S = Success (S), F = Failure. Find all requested sets.
S (sample space) = {SS, SF, FS, FF}

A = at least one packet succeeds = {SS, SF, FS}
B = both packets fail = {FF}
A′ = complement of A = {FF} (everything not in A)
A ∪ B = A or B or both = {SS, SF, FS, FF} = S
A ∩ B = A and B both = {} = ∅

A and B are mutually exclusive because A ∩ B = ∅.
(You can't simultaneously have "at least one success" and "both fail.")
§11 Counting Rules — The Multiplication Rule

Before computing probabilities, you often need to count how many possible outcomes exist. The multiplication rule is the foundation of all counting.

Multiplication Rule If operation 1 can be done in n₁ ways, operation 2 in n₂ ways, ..., operation k in nₖ ways,
then all operations together can be performed in:

n₁ × n₂ × n₃ × ··· × nₖ total ways
Example 1 — Defective Items
3 items, each can be Defective (D) or Non-defective (N). How many sample points?
Each item has 2 outcomes. Three decisions.
2 × 2 × 2 = 8 sample points
Example 2 — Password System
Password: 3 lowercase letters (26 each) + 2 digits (10 each). How many possible passwords?
3 letter positions × 2 digit positions
26 × 26 × 26 × 10 × 10 = 17,576,000
Example 3 — Three-Digit Codes
How many 3-digit codes using digits 0–9? (a) Without restriction (b) Without repetition?
(a) Each digit: 10 choices each
(b) No repetition: 10 choices for 1st, 9 for 2nd, 8 for 3rd
(a) 10 × 10 × 10 = 1,000  |  (b) 10 × 9 × 8 = 720
§12 Permutations

A permutation is an arrangement of objects where order matters. Swapping two items creates a different permutation.

📌 The Key Question for Permutations"Does the order/sequence matter?" If yes → use permutations. Example: A podium finish (1st, 2nd, 3rd place) is a permutation because Gold ≠ Silver.
Theorem 1 — Permutations of n distinct objects Number of ways to arrange ALL n distinct objects:
n! = n × (n−1) × (n−2) × ··· × 2 × 1

Number of ways to arrange n distinct objects taken r at a time:
P(n,r) = n! / (n−r)!
Example — Class Presentations
6 students presenting projects. How many possible presentation orders?
All 6 arranged in sequence: n=6, r=6
P(6,6) = 6! = 6×5×4×3×2×1 = 720 orders
Example — Assigning Interns
12 interns, assign 4 to testing roles (each assigned to a specific step). How many ways?
Order matters (Step 1 ≠ Step 2 ≠ ...). n=12, r=4
P(12,4) = 12! / (12−4)! = 12!/8! = 12×11×10×9 = 11,880

Theorem 2 — Permutations with Identical Objects

When some items are identical, swapping them doesn't create a new arrangement. We divide by the factorial of each group of identical items.

Theorem 2 n objects, where n₁ are of type 1, n₂ of type 2, ..., nₖ of type k:

Distinct arrangements = n! / (n₁! × n₂! × ··· × nₖ!)
Example
Binary strings of length 8 with exactly 5 ones and 3 zeros. How many distinct strings?
n=8, five 1s (n₁=5), three 0s (n₂=3)
8! / (5! × 3!) = 40320 / (120×6) = 40320/720 = 56 distinct strings
§13 Combinations & Partitions

Combinations — When Order Doesn't Matter

A combination is a selection of items where order does NOT matter. Choosing persons A, B, C is the same as choosing C, B, A.

📌 Permutation vs. CombinationPermutation: "Who gets 1st place, 2nd place, 3rd place?" (order matters)
Combination: "Which 3 students form a committee?" (order doesn't matter)
Theorem 4 — Combinations Number of ways to choose r items from n distinct items (order ignored):

C(n,r) = n! / [r! × (n−r)!]

Also written as: ⁿCᵣ or (n choose r) or C(n,r)
Example — Selecting Servers
A database admin must choose 5 servers from 20 to perform a task. How many possible selections?
Order doesn't matter (any 5 are equivalent). n=20, r=5
C(20,5) = 20! / (5! × 15!) = (20×19×18×17×16) / (5×4×3×2×1) = 1,860,480/120 = 15,504

Theorem 3 — Partitions (Distributing into Groups)

A partition divides n objects into r distinct groups of sizes n₁, n₂, ..., nᵣ. The internal order within each group does NOT matter.

Theorem 3 — Multinomial Coefficient ( n / n₁, n₂, ..., nᵣ ) = n! / (n₁! × n₂! × ··· × nᵣ!)
where n₁ + n₂ + ··· + nᵣ = n
Example — Distributing Database Files
12 data files to be distributed across 3 servers, 4 files each. How many ways?
n=12, three groups of n₁=4, n₂=4, n₃=4
12! / (4! × 4! × 4!) = 479,001,600 / (24×24×24) = 479,001,600/13,824 = 34,650

§14 Defining Probability — Axioms & Rules

Probability of an Event

The probability of event A, written P(A), is the sum of the probabilities (weights) of all sample points in A. Every valid probability must satisfy three axioms:

Axiom 1
Non-negativity
0 ≤ P(A) ≤ 1
Probability cannot be negative or greater than 1.
Axiom 2
Certainty
P(S) = 1
The probability that SOMETHING in the sample space happens is 1.
Axiom 3
Additivity
For mutually exclusive events:
P(A₁ ∪ A₂ ∪ ···) = P(A₁) + P(A₂) + ···

Equally Likely Outcomes

When all N outcomes are equally likely (e.g., fair dice, fair coins, random selection), the probability of event A is simply:

Classical Probability Formula P(A) = n / N

where n = number of outcomes in A
and N = total outcomes in S
Example — Fair Coin Tossed Twice
S = {HH, HT, TH, TT}. What is P(at least 1 head)?
Event A = {HH, HT, TH} — 3 outcomes contain at least 1 head. Total N = 4.
P(A) = 3/4 = 0.75
Card Examples — Standard 52-Card Deck
Deck: 4 suits × 13 cards each = 52 cards total. Find:
(a) P(Ace) → 4 aces in 52 cards
(b) P(King of Hearts) → 1 specific card
(c) P(not an Ace) → 52−4 = 48 non-Aces
(a) 4/52 = 1/13 ≈ 0.077  |  (b) 1/52 ≈ 0.019  |  (c) 48/52 = 12/13 ≈ 0.923
§14b Probability in Fraction Form

Probability can be expressed in three equivalent forms. Exams often give or ask for the fraction form — especially when the sample space involves whole counts like cards, dice, students, or items drawn from a group.

The Three Forms of Probability Fraction: P(A) = n / N (favorable outcomes over total outcomes)
Decimal: P(A) = n ÷ N (divide the fraction)
Percentage: P(A) = (n / N) × 100%
💡 Always simplify your fraction.4/52 → divide both by 4 → 1/13. 12/52 → divide both by 4 → 3/13. Unsimplified fractions are usually marked wrong or incomplete. Find the GCD (Greatest Common Divisor) of numerator and denominator and divide both by it.

How to Simplify a Probability Fraction — Step by Step

  1. Write the fraction as favorable outcomes / total outcomes.
  2. Find the GCD (Greatest Common Divisor) of numerator and denominator. Ask: "What is the largest number that divides both evenly?"
  3. Divide both numerator and denominator by the GCD.
  4. Check that the resulting fraction can't be reduced further.
Example — Simplify 18/24
GCD of 18 and 24?
Factors of 18: 1, 2, 3, 6, 9, 18
Factors of 24: 1, 2, 3, 4, 6, 8, 12, 24
Largest common factor = 6
18 ÷ 6 = 3  |  24 ÷ 6 = 4
18/24 = 3/4

Applying All Probability Rules in Fraction Form

Classical Formula

Example — Die Roll
A fair 6-sided die is rolled. Find P(even number).
Even numbers: {2, 4, 6} → n = 3
Total outcomes: {1, 2, 3, 4, 5, 6} → N = 6
P(even) = 3/6 = 1/2
Example — Drawing from a Group
A bag has 5 red, 3 blue, and 2 green balls. One is drawn at random. Find P(blue).
Favorable: 3 blue balls. Total: 5 + 3 + 2 = 10 balls.
P(blue) = 3/10  |  decimal: 0.3  |  percentage: 30%

Complement Rule in Fraction Form

Complement P(A') = 1 − P(A) = N/N − n/N = (N − n) / N
Example
From the same bag (5R, 3B, 2G). Find P(not blue).
P(blue) = 3/10
P(not blue) = 1 − 3/10 = 10/10 − 3/10 = 7/10
P(not blue) = 7/10 = 0.7

Additive Rule in Fraction Form

Example — Cards (Diamond OR Ace)
From a 52-card deck: P(diamond OR ace)?
P(diamond) = 13/52
P(ace) = 4/52
P(ace of diamonds) = 1/52 ← counted in both, must subtract
P(D ∪ A) = 13/52 + 4/52 − 1/52 = 16/52
16/52 = 4/13 ≈ 0.308

Conditional Probability in Fraction Form

Conditional P(B|A) = P(A ∩ B) / P(A) → in fraction form, divide the two fractions:
(a/N) ÷ (b/N) = a/b (the N cancels)
Example — Reduced Sample Space
From a group of 10 students: 6 pass, 4 fail. Of the 6 who pass, 2 are female. Given a student passed, what is P(female)?
P(pass) = 6/10
P(female AND pass) = 2/10
P(female | pass) = P(F ∩ pass) / P(pass) = (2/10) / (6/10)
= 2/10 × 10/6 = 2/6 = 1/3 ≈ 0.333

Multiplication Rule in Fraction Form

Example — Drawing Without Replacement
A bag has 4 red and 6 blue balls. Two are drawn without replacement. Find P(both red).
P(1st red) = 4/10
P(2nd red | 1st red) = 3/9 ← one red removed, 9 remain
P(both red) = 4/10 × 3/9 = 12/90
12/90 = 2/15 ≈ 0.133
Example — Drawing With Replacement (Independent)
Same bag. Two balls drawn WITH replacement. Find P(both red).
P(1st red) = 4/10 = 2/5
P(2nd red) = 4/10 = 2/5 ← same, because ball is returned
P(both red) = 2/5 × 2/5 = 4/25
P(both red) = 4/25 = 0.16

Fraction Form with Counting Rules

Example — Combination Probability
A committee of 3 is chosen from 5 men and 4 women (9 total). What is P(all 3 are women)?
Favorable: C(4,3) = 4! / (3! × 1!) = 4
Total: C(9,3) = 9! / (3! × 6!) = (9×8×7)/(3×2×1) = 84
P(all women) = 4/84 = 1/21 ≈ 0.048
⚠️ Common Fraction Mistakes to Avoid (1) Forgetting to simplify — always reduce to lowest terms.
(2) Adding fractions with different denominators without finding a common denominator first: 1/4 + 1/3 ≠ 2/7. It equals 3/12 + 4/12 = 7/12.
(3) Dividing fractions — remember to multiply by the reciprocal: (2/10) ÷ (6/10) = 2/10 × 10/6 = 20/60 = 1/3.
(4) Confusing "without replacement" (dependent events, denominator shrinks) with "with replacement" (independent events, denominator stays the same).
§15 Additive Rule (Union of Events)

When you want the probability of A or B (or both) occurring, use the additive rule.

General Additive Rule P(A ∪ B) = P(A) + P(B) − P(A ∩ B)

Special case — if A and B are mutually exclusive (A ∩ B = ∅):
P(A ∪ B) = P(A) + P(B)
⚠️ Why subtract P(A ∩ B)?When you add P(A) + P(B), the outcomes in BOTH A and B get counted twice. Subtracting P(A ∩ B) corrects this double-count.
Card Example — Diamond OR Ace
From a 52-card deck: P(diamond card OR ace)?
A = diamond → 13/52
B = ace → 4/52
A ∩ B = ace of diamonds → 1/52 (counted in both!)
P(A ∪ B) = 13/52 + 4/52 − 1/52 = 16/52
P(Diamond OR Ace) = 16/52 = 4/13 ≈ 0.308
Card Example — Diamond OR Heart (Mutually Exclusive)
P(diamond OR heart)? (A card cannot be both)
A = diamond (13/52), C = heart (13/52), A ∩ C = ∅
P(A ∪ C) = 13/52 + 13/52 = 26/52 = 1/2 = 0.5
Machine Failure Example
P(A) = 0.2, P(B) = 0.3, P(A ∩ B) = 0.1. What is P(at least one fails)?
P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
P(A ∪ B) = 0.2 + 0.3 − 0.1 = 0.4

Partition of a Sample Space

A collection of events {A₁, A₂, ..., Aₙ} is a partition of S if they are mutually exclusive AND their union equals S. The sum of their probabilities always equals 1.

P(A₁) + P(A₂) + ··· + P(Aₙ) = P(S) = 1
Programming Language Preferences
P(Python)=0.30, P(Java)=0.25, P(C++)=0.20, P(JavaScript)=0.15. P(one of these)?
These are mutually exclusive (a student prefers exactly one). They don't cover everyone (sum < 1), so it's not a partition of S.
P(Python ∪ Java ∪ C++ ∪ JS) = 0.30 + 0.25 + 0.20 + 0.15 = 0.90
§16 Complement Rule

If P(A) is the probability that A happens, then the probability A does NOT happen is its complement P(A′).

Complement Rule P(A) + P(A′) = 1

Therefore: P(A′) = 1 − P(A)
💡 When to Use the ComplementUse the complement rule when calculating P(A) directly is hard, but P(A′) is easy. "At least one" problems are the most common case: P(at least one) = 1 − P(none).
Example 1 — Cards
What is P(not an Ace) from a 52-card deck?
P(Ace) = 4/52 = 1/13
P(not Ace) = 1 − 1/13 = 12/13 ≈ 0.923
Example 2 — Bug-Free Program
P(program has at least one bug) = 0.35. What is P(no bugs)?
P(no bugs) = 1 − 0.35 = 0.65
Example 3 — Web Server Requests
P(100 req) = 0.08, P(200) = 0.15, P(300) = 0.25, P(400) = 0.30, P(500) = 0.12, P(600+) = 0.10. Find P(at least 200 requests).
P(at least 200) = 1 − P(fewer than 200) = 1 − P(exactly 100)
P(at least 200) = 1 − 0.08 = 0.92
§17 Conditional Probability

Conditional probability is the probability that event B occurs given that event A has ALREADY occurred. Knowing A has happened changes the sample space — we now only consider outcomes where A is true.

Definition P(B | A) = P(A ∩ B) / P(A)    provided P(A) > 0

Read as: "The probability of B GIVEN A"
📌 IntuitionWhen A has occurred, the sample space "shrinks" to just A. Conditional probability asks: among all outcomes in A, what fraction also belong to B?
CS Student Example
900 CS students: 500 prefer Python, 400 prefer Java. Of those, 460 Python students and 140 Java students are in Research (600 total in Research). (a) P(student is in Research)? (b) P(student prefers Python | student is in Research)?
(a) Research students = 600 out of 900
(b) P(Python ∩ Research) = 460/900. P(Research) = 600/900.
P(Python | Research) = (460/900) ÷ (600/900) = 460/600
(a) 600/900 = 2/3 ≈ 0.667  |  (b) 460/600 ≈ 0.767
Server Performance Example
P(R) = 0.85 (responds within 1 sec), P(C) = 0.80 (correct result), P(R ∩ C) = 0.75. Find (a) P(C|R), (b) P(R|C), (c) P(C|R′).
(a) P(C|R) = P(R ∩ C)/P(R) = 0.75/0.85
(b) P(R|C) = P(R ∩ C)/P(C) = 0.75/0.80
(c) P(R′) = 1 − 0.85 = 0.15. P(C ∩ R′) = P(C) − P(C ∩ R) = 0.80 − 0.75 = 0.05
P(C|R′) = 0.05/0.15
(a) 0.75/0.85 ≈ 0.882  |  (b) 0.75/0.80 = 0.9375  |  (c) 0.05/0.15 ≈ 0.333
§18 Multiplication Rule for Probability

The multiplication rule gives the probability of both events A and B occurring together (their intersection).

Multiplication Rule P(A ∩ B) = P(A) × P(B | A)

Equivalently: P(A ∩ B) = P(B) × P(A | B)
Example — Two Cards Without Replacement
What is P(first card = King of Hearts AND second card = Ace), drawing without replacement?
A = first card is King of Hearts → P(A) = 1/52
B|A = second card is an Ace, given 51 cards remain, all 4 Aces still there → P(B|A) = 4/51
P(A ∩ B) = (1/52) × (4/51) = 4/2652 = 1/663 ≈ 0.00151
§19 Independent vs. Dependent Events

Definition of Independence

Two events are independent if knowing that one occurred gives no information about whether the other will occur — it doesn't change the probability.

Independence Condition A and B are independent if and only if:
P(B|A) = P(B)    OR equivalently    P(A ∩ B) = P(A) × P(B)
Coin Toss — Independent Events
A = first toss is Head, B = second toss is Head. Are A and B independent?
P(A) = 2/4 = 1/2, P(B) = 2/4 = 1/2
A ∩ B = {HH} → P(A ∩ B) = 1/4
P(B|A) = P(A ∩ B)/P(A) = (1/4)/(1/2) = 1/2 = P(B)
P(B|A) = P(B) → A and B are INDEPENDENT. The first toss doesn't affect the second.
Card Draw — Dependent Events
A = first card is King of Hearts. B = second card is an Ace. Drawing WITHOUT replacement. Independent?
P(B) = 4/52 = 1/13 (before drawing anything)
P(B|A) = 4/51 (4 aces remain among 51 cards after King of Hearts is drawn)
1/13 ≠ 4/51
P(B) ≠ P(B|A) → A and B are DEPENDENT. Removing a card changes what's left.

Multiplication Rule for Independent Events

If A and B are Independent P(A ∩ B) = P(A) × P(B)
Example — Two Virtual Machines
Each VM is up 99.5% of the time independently. P(both are up simultaneously)?
P(A) = 0.995, P(B) = 0.995. Independent events.
P(A ∩ B) = 0.995 × 0.995 = 0.990025 ≈ 99%
Example — Memory Leak + Crash
P(memory leak) = 0.1. P(system crashes | memory leak) = 0.7. P(both occur)?
These are DEPENDENT. Use: P(A ∩ B) = P(A) × P(B|A)
P(leak ∩ crash) = 0.1 × 0.7 = 0.07

§20 Total Probability & Bayes' Rule

The Law of Total Probability

Sometimes you can't find P(A) directly. But if you can partition the sample space into groups B₁, B₂, ..., Bₖ, you can calculate P(A) by summing the probability of A within each group.

Law of Total Probability (Rule of Elimination) If B₁, B₂, ..., Bₖ is a partition of S (mutually exclusive, union = S), then:

P(A) = Σ P(Bᵢ) × P(A | Bᵢ)
= P(B₁)P(A|B₁) + P(B₂)P(A|B₂) + ··· + P(Bₖ)P(A|Bₖ)
Special Research Project Example
900 CS students: 500 prefer Python, 400 prefer Java. 20 Python students and 15 Java students are in a special project. P(a random student is in the special project)?
B₁ = prefers Python → P(B₁) = 500/900
B₂ = prefers Java → P(B₂) = 400/900
P(A|B₁) = 20/500 (20 of 500 Python students are in project)
P(A|B₂) = 15/400 (15 of 400 Java students are in project)

P(A) = (500/900)(20/500) + (400/900)(15/400)
= 20/900 + 15/900 = 35/900
P(A) = 35/900 ≈ 0.0389 ≈ 3.89%
💡 Total Probability in Plain EnglishThink of it as a weighted average. "What fraction of Python students are in the project?" + "What fraction of Java students?" weighted by how many students are in each group.
📌 Bayes' Rule (Extension)After computing total probability, Bayes' Rule lets you "reverse" the conditional probability — going from P(A|Bᵢ) to P(Bᵢ|A):

P(Bᵢ|A) = P(Bᵢ) × P(A|Bᵢ) / P(A)

Example: "Given that a student IS in the project, what's the probability they prefer Python?"
P(Python | in project) = P(Python) × P(in project | Python) / P(in project)
= (500/900)(20/500) / (35/900) = (20/900) / (35/900) = 20/35 ≈ 0.571
